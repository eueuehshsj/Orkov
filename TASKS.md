# Enter to Orkov — 1단계 프로토타입 작업 목록

> 기반: Godot 4.6 / 탑다운 익스트랙션 슈터 / 검증 목표: 입장→전투→회수→귀환→강화 루프가 재미있는가

---

## 우선순위 1 — 프로젝트 기반

게임 루프의 뼈대. 이게 없으면 나머지를 붙일 곳이 없다.

- [x] **폴더 구조 정리**
  - `scenes/`, `scripts/`, `assets/`, `resources/` 기본 디렉토리 생성
- [x] **데이터 구조 정의** (`Resource` 클래스)
  - `WeaponData.gd` — 무기 스탯(데미지, 사거리, 쿨다운, 명중률)
  - `EnemyData.gd` — 적 스탯(체력, 공격력, 이동속도, 탐지 범위)
  - `UpgradeData.gd` — 강화 항목(데미지+, 방어+ 각 3단계)
- [x] **GameManager 싱글톤**
  - 씬 전환 함수 (`go_to_stage()`, `go_to_base()`)
  - 전역 상태 보관 (스크랩 보유량, 강화 레벨, 현재 스테이지 결과)
- [x] **씬 구조 설계**
  - `main_menu.tscn` → `stage.tscn` → `base.tscn` 전환 흐름 확정

---

## 우선순위 2 — 플레이어

전투 검증의 핵심 주체.

- [x] **탑다운 이동** (`CharacterBody2D`)
  - WASD 8방향 이동, `move_and_slide()`
  - 마우스 방향으로 회전 (데프 드레드 포탑 느낌)
- [x] **카메라** (`Camera2D`)
  - 플레이어 추적, 부드러운 이동 (`position_smoothing`)
- [x] **체력 시스템**
  - `max_hp`, `current_hp`, `take_damage()`, `die()` 함수
  - 사망 시 GameManager에 실패 신호 전달
- [x] **히트박스 / 헐트박스**
  - `CollisionShape2D` 분리 (물리 충돌용 vs 피격 판정용)

---

## 우선순위 3 — 무기

플레이어 다음으로 전투 검증에 직결된다.

- [x] **근접 클로** (메인 딜러)
  - 공격 키(LMB) 입력 시 전방 짧은 부채꼴 범위 판정
  - `Area2D` + `CollisionShape2D`로 히트박스 구현
  - 높은 데미지, 짧은 쿨다운
  - 공격 애니메이션 (없으면 임시 Tween으로 대체)
- [x] **슬러거** (원거리 보조)
  - RMB 또는 E 키로 발사
  - 총알 씬 (`Bullet.tscn`) 인스턴스 생성, 직선 이동
  - 명중률 보정: 발사 각도에 랜덤 오프셋 추가
  - 낮은 DPS (클로의 보조 역할 확인용)
- [x] **무기 슬롯 관리**
  - 슬롯 1개, 현재 무기 참조 유지
  - 무기 교체는 1단계 미구현 (클로+슬러거 동시 보유)

---

## 우선순위 4 — 적 AI

전투 루프의 반대편. 3개 적 유형 모두 필요.

- [x] **Enemy 베이스 클래스** (`enemy_base.gd`)
  - 체력, 공격력, 이동속도 변수
  - `take_damage()`, `die()` (스크랩 드랍 포함)
  - 상태머신: `PATROL → CHASE → ATTACK → DEAD`
- [x] **라이플병** (기본 원거리)
  - 순찰 경로(웨이포인트) 이동
  - 탐지 범위(`Area2D`) 진입 시 CHASE
  - 사격 범위 내 정지 후 단발 사격
- [x] **점사 사격병** (원거리 변형)
  - 라이플병과 동일 구조
  - 사격 패턴만 다름: 0.15초 간격 3연발 후 긴 쿨다운
- [x] **엘리트** (강화 버전)
  - 라이플병 데이터에서 공격력 ×1.3, 체력 ×1.3
  - 처치 시 목표 아이템 드랍 (`QuestItem.tscn`)
  - 별도 비주얼(색상 오버레이 등)로 구분
- [x] **내비게이션**
  - `NavigationRegion2D` + `NavigationAgent2D`로 장애물 우회 이동

---

## 우선순위 5 — 스테이지 & 목표 시스템

루프의 흐름을 완성하는 구조.

- [x] **맵 제작**
  - `TileMap`으로 단일 맵 (벽, 바닥, 엄폐물 기본 배치)
  - 충돌 레이어 설정
- [x] **목표 시스템** (`QuestManager.gd`)
  - 메인 목표: 엘리트 처치 후 아이템 회수 여부 추적
  - 보조 목표: 전 적 섬멸 여부 추적
  - 목표 완료 신호 → HUD 갱신
- [x] **추출 포인트**
  - `Area2D`로 추출 존 구현
  - 메인 목표 완료 전 진입 시 경고 메시지
  - 메인 목표 완료 후 진입 시 귀환 시퀀스 시작
- [x] **아이템 드랍**
  - 엘리트 처치 시 `QuestItem` 씬 스폰
  - 플레이어가 접촉하면 자동 획득 + 목표 달성 처리

---

## 우선순위 6 — 자원 & 강화

루프의 후반부. 전투 검증 후 바로 이어져야 "루프" 느낌이 난다.

- [ ] **스크랩 수집**
  - 적 처치 시 스크랩 오브젝트 스폰
  - 플레이어 접촉 시 자동 획득 또는 귀환 시 자동 정산 (선택)
  - 귀환 실패(사망) 시 스크랩 전량 손실 처리
- [ ] **귀환 정산 화면** (`base.tscn`)
  - 획득 스크랩, 목표 달성 여부 표시
  - "다음 스테이지" / "강화하기" 버튼
- [ ] **강화 시스템**
  - 데미지 강화 3단계 (스크랩 비용 증가)
  - 방어 강화 3단계
  - 강화 레벨 GameManager에 저장, 스테이지 진입 시 플레이어에 적용

---

## 우선순위 7 — UI / HUD

플레이어가 상황을 읽는 수단. 기능 검증 후 마지막에 다듬는다.

- [ ] **인게임 HUD**
  - 체력 바
  - 현재 목표 텍스트 (메인/보조 달성 여부)
  - 보유 스크랩 수
- [ ] **추출 포인트 표시**
  - 미니맵 또는 화면 내 방향 화살표
- [ ] **사망 / 귀환 오버레이**
  - 사망 시: "임무 실패 — 스크랩 손실" 팝업
  - 귀환 시: 페이드 아웃 후 정산 화면 전환
